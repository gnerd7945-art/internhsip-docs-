gstream is pipeline base architecture that offload heavy lifting-decoding(h264/h265), scaling and colorspace conversion task to dedicated gpu ,making cpu free for other CV logic. 
**Structure of it**:-
1) elements- individual machines, each does one job( v412src) grab camera feed, (nvv412decoder) decodes video usign nvidia hardware.*filters* elements which have both source and sink pad. 
2) pads- connceting point through  which data flow. source pad: data leaves element from here, sink pad: data enters element from here. 
3) caps - lightweight object that describes format and properties  of video/audio that can flow thrruogh pads.like what media(video/audio, what codec/encoding used , what resolution etc. 
4) pipeline- container that holds these elements and manage there state. 
**appsink** is last element in pipeline  which takes gstream buffer and wraps it such that it can be handed to cpp. 

emit-signals = True  in appsink allow us to  use   callback fucntion, which is more efficeint then opencv's loop and wait.In  opencv cpu wait for hardware ( sits idel). 

-> Gstream is build on GOobject( c version cpp classes) 
-> Gstream object can be extended using GOobject inheritance method. Gslice allocator is used for high performance. 
-> bin is collection of elemnts, can treat as an element itself. 
-> events are object sent between elements.
-> messages  are object posted by elements on pipeline's message bus. 
-> queries: allow elements to ask information from peer elements. 
-> gst-inspect-1.0 element_name/ factory_name // will give provide generic info.

   GstElementFactor* factory = gst_element_factory_find('some_element"); // build the blueprint for some_element

   printf("THIS FACTORY CREATES": type); // print type 
   GstElement* new1 = gst_element_factory_create(factory,"my_decoder");new1 points to actual instance, factory is the blueprint, gst_element_factory_create manually build the element.

4 STATES IN GSTREAMING:-
1) GST_STATE_NULL: gstelement* struct is just sitting in memory with no camera acess or gpu usage. THis state occurs when element first gets created or while shutting down app.
2) GST_STATE_READY : The element eg: (v412src) checks if camera is actually plugged in,it claims hardware and no other program  can access camera.here, we check if camera path correct or not.
3) GST_STATE_PAUSED(ram meory allocation and pushing): pipeline starts moving data until it hits sink. appsink will receive first frame and hold it.the moment we hit play data is already sitting at finish line.
4) GST_STATE_PLAYING: clock starts running, buffer flow continously.
* we can change state using -> gst_element_set_state(pipeline,GST_STATE_PLAYING);*
*OpenCV tries to do all this in once thus cpu stuck waiting. Here we can set state to pause making cpu ready to digest data immediately.*
